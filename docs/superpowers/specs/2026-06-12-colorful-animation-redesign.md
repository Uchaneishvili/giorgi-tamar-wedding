# 🌸 „აყვავებული ბაღი" — ფერისა და ანიმაციის რედიზაინი

**პროექტი:** გიორგი & თამარი — საქორწინო მოწვევა
**თარიღი:** 2026-06-12
**მიზანი:** არსებული, თავშეკავებული მინიმალისტური მოწვევის გადაქცევა **მაქსიმალურად ფერად და ცოცხალ** გამოცდილებად — ისე, რომ ელეგანტობა და წაკითხვადობა არ დაიკარგოს.
**სტეკი:** Vanilla HTML / CSS / JS (framework-ის გარეშე — როგორც ახლა).

> **დეველოპერისთვის TL;DR:** ვამატებთ 5 ახალ ფერს custom property-ებად, თითო სექციას ვაძლევთ საკუთარ ფერად ტონალობას, და ვამდიდრებთ ანიმაციებს (ნაწილაკები, აყვავება, კონფეტი, გრადიენტული ტექსტი). სტრუქტურა, სექციები და HTML ჩონჩხი **არ იცვლება** — ეს გაუმჯობესებაა, არა გადაწერა.

## 0. პრინციპები (წაიკითხე პირველ რიგში)

1. **Performance first.** ანიმაცია მხოლოდ `transform`-სა და `opacity`-ზე. layout-ის (`width`, `top`, `margin`) ანიმაცია — აკრძალულია. `will-change` მხოლოდ აქტიური ანიმაციის დროს, შემდეგ მოიხსნას.
2. **`prefers-reduced-motion: reduce`** — ყველა ახალი ანიმაცია უნდა ითიშებოდეს ან მკვეთრად მცირდებოდეს. პატერნი უკვე არსებობს `styles.css`-ში (ბოლო `@media` ბლოკები) — გააგრძელე იგივე სტილით. SMIL ანიმაციები JS-ით იშლება (`js/main.js:5`) — ახალი SMIL/canvas ანიმაციებიც იქ უნდა გაითიშოს.
3. **ფერი = თემა.** ყველა ფერი `:root`-ში custom property-დ. სექციებს ფერი ენიჭებათ ლოკალური `--section-tint` ცვლადით, არა hardcoded მნიშვნელობით.
4. **ნაწილაკების ლიმიტი.** მობილურზე max ~12-15 ნაწილაკი, დესკტოპზე max ~25-30. canvas სჯობს ბევრ DOM ელემენტს.
5. **Vanilla.** არანაირი ბიბლიოთეკა. ანიმაციები — CSS keyframes + IntersectionObserver + requestAnimationFrame (პატერნები უკვე გვაქვს `reveal.js`-ში).
6. **ელეგანტობის ზღვარი.** „ფერადი" ≠ „ჭრელი". ფერები ხარისხიანი, ცოტათი დადუნებული (muted), არა ნეონი. ფონი რჩება კრემისფერი.

## 1. ფერების სისტემა

### 1.1 ახალი ცვლადები — `css/styles.css` → `:root`

არსებულ ცვლადებს (`--accent`, `--sage`) ვამატებთ ბაღის ბუკეტს:

```css
:root {
  /* --- არსებული (რჩება) --- */
  --bg:       #FAFAF7;
  --bg-alt:   #F3EFE7;
  --ink:      #0F0F0F;
  --ink-soft: #5C5852;
  --accent:   #C97B5C;  /* ტერაკოტა — primary */
  --sage:     #7A8B6A;  /* ფოთლები / ღეროები */
  --line:     #E8E2D8;

  /* --- 🆕 ბაღის პალიტრა --- */
  --bloom-saffron: #E8A33D;  /* ზაფრანა / მზესუმზირა */
  --bloom-blush:   #E89B8B;  /* ვარდისფერი blush */
  --bloom-fuchsia: #D45D79;  /* ფუქსია / მაჟენტა */
  --bloom-peri:    #7BA6C9;  /* პერივინკლი / ცისფერი */
  --bloom-plum:    #6B4E71;  /* ქლიავისფერი — სიღრმე */

  /* --- 🆕 გრადიენტები (გადანაწილებადი) --- */
  --grad-warm:  linear-gradient(135deg, var(--accent), var(--bloom-saffron));
  --grad-bloom: linear-gradient(135deg, var(--bloom-fuchsia), var(--bloom-blush));
  --grad-fresh: linear-gradient(135deg, var(--sage), var(--bloom-peri));
}
```

### 1.2 სექციური ტონალობა

თითო სექციას ენიჭება „გასაღები ფერი" (`--section-tint`), რომელიც გამოიყენება ფონის ნაზ ჰალოზე, ორნამენტებზე და hover-ებზე. სქროლისას ვიზიტორი „ფერების მოგზაურობას" გადის:

| სექცია | ტონალობა | განცდა |
|---|---|---|
| Hero | მრავალფერი მესი (თბილი→გრილი) | აღფრთოვანება |
| Welcome | blush + sage | სითბო, ინტიმი |
| Countdown | saffron | ენერგია, მოლოდინი |
| Events | peri + sage | სიმშვიდე, სიწმინდე |
| RSVP | fuchsia + saffron | აღნიშვნა, ზეიმი |
| Footer | plum → accent გრადიენტი | დასასრულის სიდინჯე |

იმპლემენტაცია: თითო `<section>`-ს მიეცეს მოდიფიკატორი (მაგ. `.countdown { --section-tint: var(--bloom-saffron); }`) და ფონის ჰალო ერთი საერთო წესით აიგოს.

## 2. სექცია-სექციის სპეციფიკაცია

### 2.1 ✉️ კონვერტის ინტრო (`#envelope-intro`, `js/envelope.js`)

**ფერი:**
- ცვილის ბეჭედი (`.envelope__seal circle`) — `--accent`-დან გრადიენტზე (`--grad-warm`). მონოგრამა `G&T` რჩება კრემისფერი.
- კონვერტის სარჩული (გახსნისას ჩანს) — ნაზი `--bloom-blush` ელფერი.

**ანიმაცია — გასახსნელი მომენტი (მთავარი „wow"):**
- ბეჭდის გახსნისთანავე **ფურცლების/კონფეტის აფეთქება** ცენტრიდან — 18-24 ნაწილაკი ბაღის 5 ფერში, აფრინდება და ცვივა (gravity + drift).
- იმპლემენტაცია: დროებითი `<canvas>` overlay-ზე, requestAnimationFrame, ~1.2წმ შემდეგ თვითგაუქმება. ან მსუბუქი CSS — 20 აბსოლუტურად პოზი+ `@keyframes confetti-fall`, `animationend`-ზე წაიშალოს.
- `envelope:opened` event უკვე იფრქვევა (`js/main.js:58`) — კონფეტი ამ event-ს მოუსმინოს.

**Reduced-motion:** კონფეტი არ ჩაირთვება; ბეჭედი უბრალოდ ქრება (არსებული fallback).

### 2.2 🏛️ Hero (`#hero`, `css/styles.css:159+`)

ეს არის ფერის მთავარი სცენა. სამი ფენა:

**ფენა 1 — გრადიენტ-მესი ფონი (`.hero::before`):**
- არსებული 4-წერტილიანი radial-gradient მესი (`styles.css:174`) გავამდიდროთ 6 წერტილამდე, ბაღის ფერების ჩართვით (`--bloom-fuchsia`, `--bloom-peri`, `--bloom-saffron`), alpha ~0.25-0.35.
- `hero-glow` keyframes (`styles.css:189`) გავაგრძელოთ; დავამატოთ ნაზი `hue-rotate` ან opacity-pulse, რომ ფერები „სუნთქავდეს". ციკლი ნელი — 30-40წმ.

**ფენა 2 — 🆕 ამავალი ფურცლების ნაწილაკები:**
- 15-25 პატარა ფურცელი/ყვავილი ნელა ცაში ადის, ოდნავ ირხევა (sine drift), მუდმივი ციკლი.
- `<canvas>` `position:absolute; inset:0; z-index:0; pointer-events:none` hero-ში. ან მსუბუქი CSS — რამდენიმე SVG ფურცელი `@keyframes float-up`-ით (staggered delay).
- ფერები ბაღის პალიტრიდან, opacity ~0.4-0.7.

**ფენა 3 — არქის ვაზი (`.arch__vine`, `styles.css:1025+`):**
- ამჟამად ვაზი ერთფერია (accent + sage). **დავაფეროთ ყვავილები** — თითო `.vine__flower`-ს ვაძლევთ სხვადასხვა ფერს `nth-child`/`--i` ლოგიკით (saffron, blush, fuchsia როტაციით).
- staggered „pop" entrance უკვე არსებობს (`styles.css:1030`) — შევინარჩუნოთ, ფერი დავამატოთ.

**სახელები (`.hero__names`):**
- `--accent`-ის ნაცვლად ამპერსანდს (`.hero__amp`) გრადიენტი (`--grad-bloom`, `background-clip:text`).
- სახელებზე — per-letter cascade უკვე არსებობს (`main.js:29`); დავამატოთ ნაზი ფერის გადავლება (sweep) დასრულების შემდეგ — `background-clip:text` + მოძრავი გრადიენტი, ან subtle accent underline draw.

**Reduced-motion:** ნაწილაკები ითიშება; გრადიენტი სტატიკურია; ვაზი/სახელები მაშინვე ჩანს (პატერნი არსებობს).

### 2.5 📍 Events (`#events`, `styles.css:812+`)

**Reduced-motion:** SMIL/canvas ანიმაცია იშლება (`main.js:5` უკვე ფარავს), სცენა სტატიკურია.

### 2.8 🖱️ გლობალური მიკრო-ინტერაქციები

- **კურსორის რგოლი** (`.cursor-ring`, `styles.css:790`, `js/cursor.js`) — `--ink`-ის ნაცვლად სექციის `--section-tint`-ზე იცვლებოდეს (კონტექსტური ფერი). hover-ზე ოდნავ შეიფეროს.

- **Scroll-driven ფურცლების ცვენა (არასავალდებულო, nice-to-have):** მთელ გვერდზე იშვიათი, ნაზი ფურცლების ცვენა ფონზე. გადატვირთვის რისკი — ბოლოს, თუ დრო იქნება.

## 3. ფაილების სტრუქტურა (რა იცვლება / ემატება)
